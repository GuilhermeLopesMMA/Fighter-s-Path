Decisão Visual v3.0

O jogo passa de top-down 2D para 3D isométrico com câmara fixa, inspirado directamente em Pokémon HeartGold/SoulSilver. Cenários em low-poly 3D, personagens em sprite 2D billboard ou low-poly 3D simples.

Mudança de arquitectura: 2D top-down → 3D isométrico estilo HeartGold. O sistema de combate por turnos mantém-se igual.

Referência — Pokémon HeartGold/SoulSilver

HeartGold usa câmara ortográfica em ângulo fixo sobre cenários 3D com personagens em sprite. É o equilíbrio perfeito entre profundidade visual e legibilidade de gameplay.

Antes vs Depois

❌ v2.0 — 2D Top-Down
Câmara perpendicular ao chão. Tudo flat. Tilesets sprite. Sem profundidade. Estilo clássico Game Boy — bom para alpha HTML mas limitado visualmente para Unity.
✅ v3.0 — 3D Isométrico
Câmara ortográfica em ângulo fixo. Edifícios com altura real. Ruas com profundidade. Personagens com shadow. Permite reflexo de luz, animações 3D de ambiente, câmara cinematográfica em cutscenes.

O que NÃO muda

A mudança é só visual. O sistema de gameplay mantém-se idêntico:

Sistema de Câmara

Câmara ortográfica em ângulo fixo — a base do estilo HeartGold. O jogador nunca controla a câmara directamente; ela segue o personagem suavemente com limites por zona.

Projecção
Ortográfica
Sem perspectiva. Objectos distantes têm o mesmo tamanho que os próximos. Dá o look "isométrico limpo" do HeartGold.
Ângulo
45–60° overhead
Rotação X entre 45° e 60°. Permite ver as fachadas dos edifícios e o chão simultaneamente. Testar ambos e escolher.
Rotação Y
Fixa (45° ou 0°)
Câmara não roda com o jogador. Orientação fixa tipo isométrico clássico. O personagem muda de sprite/animação consoante a direcção.
Seguimento
Cinemachine 3D
Virtual Camera com Follow no player. Damping suave. Confiner por zona de mapa. Dead zone no centro para evitar micro-movimentos.
Cutscenes
Timeline + Dolly
Para entrada em ginásios e cenas de história, a câmara anima com Cinemachine Dolly Track. Transição suave entre câmaras.
Combate
Câmara separada
Ao entrar em combate, fade-to-black e carrega cena de batalha com câmara lateral fixa — como no Pokémon.

Assets & Pipeline

Tudo low-poly com flat shading — sem texturas fotorrealistas. As cores são aplicadas directamente nos materiais (vertex colors ou materiais simples). Permite assets leves, fáceis de produzir e com identidade visual forte.

Regra de ouro: nenhum modelo deve ter mais de 500 polígonos. Edifícios simples: 50–150. Personagens: 200–500. Árvores/props: 20–80.

Estilo Visual — Flat Shading

Shading
Flat / Unlit
Cada face tem uma cor uniforme. Sem gradientes de luz suave. URP Lit com Smoothness=0 ou shader Unlit custom.
Texturas
Mínimas ou nenhumas
Cor aplicada no material. Excepções: azulejos portugueses (textura simples 64x64px) e calçada (pattern repetido).
Iluminação
Directional + Ambient
Uma só Directional Light. Ambient colour quente. Sem sombras dinâmicas pesadas — shadow maps simples apenas no player.
Paleta
Por cidade
Lisboa: ocres e azuis. Porto: cinzentos e verdes. Montijo: castanhos e azul Tejo. Cada cidade tem 8–10 cores-base.

Pipeline de Produção

1
Blockout em ProBuilder

Construir o layout de cada cidade com caixas simples directamente no Unity. Validar proporções e gameplay antes de modelar.

2
Modelação em Blender

Substituir boxes por modelos low-poly detalhados. Exportar como .fbx. Aplicar Shade Flat no Blender antes de exportar.

3
Materiais no Unity

Criar materiais URP Lit com Smoothness=0 e Metallic=0. Atribuir cores da paleta da cidade. Sem texturas excepto casos especiais.

4
Iluminação e pós-processamento

URP Global Volume com Color Grading subtil. Bloom ligeiro. Vignette leve. Depth of Field desactivado (quebra o isométrico).

5
Optimização

Static Batching em todos os edifícios. GPU Instancing nos props repetidos. Occlusion Culling activado. LOD apenas se necessário.

Unity Setup 3D Isométrico

Versão
Unity 2022 LTS
Versão estável de longa duração. Evitar Unity 6 por agora — LTS tem mais documentação e menos bugs.
Pipeline
URP — Universal
Universal Render Pipeline. Mais leve que HDRP. Suporte a shaders low-poly e pós-processamento para o estilo visual pretendido.
Câmara
Cinemachine 3D
Cinemachine Virtual Camera ortográfica. Follow no player. Confiner 3D por zona. Timeline para cutscenes.
Modelação
ProBuilder + Blender
ProBuilder para blockout rápido no Unity. Blender para modelos finais. Importação via .fbx com Shade Flat.
Animação
Animator 3D
Animator Controller com blend tree para 8 direcções (ou 4). Animações: Idle, Walk, Run, Interact. Mixamo como base.
Colisão
NavMesh + Colliders
NavMesh para pathfinding dos NPCs. Mesh Colliders simples nos edifícios. CharacterController no player.
Diálogo
Yarn Spinner
Igual ao planeado. Detecção de NPC por trigger 3D (Sphere Collider como zona de interacção).
Combate
Cena separada
LoadSceneAsync para cena de batalha. Passar dados do inimigo via ScriptableObject ou Singleton de sessão.

// PlayerController.cs public class PlayerController : MonoBehaviour { [SerializeField] float speed = 4f; CharacterController cc; Animator anim; // Isométrico: converter input para direcção de mundo readonly Vector3 isoForward = new Vector3(1,0,1).normalized; readonly Vector3 isoRight = new Vector3(1,0,-1).normalized; void Update() { float h = Input.GetAxisRaw("Horizontal"); float v = Input.GetAxisRaw("Vertical"); Vector3 move = (isoForward * v + isoRight * h).normalized; if (move.magnitude > 0.1f) { cc.Move(move * speed * Time.deltaTime); transform.rotation = Quaternion.LookRotation(move); anim.SetBool("Walking", true); } else { anim.SetBool("Walking", false); } } }

Sistema de Combate

O combate mantém-se por turnos em cena separada — igual ao protótipo HTML. A cena de batalha tem câmara lateral (side-on) como no Pokémon clássico, mesmo que o mapa seja isométrico.

A transição mapa → combate usa fade-to-black + LoadSceneAsync("BattleScene"). Os dados do inimigo passam via GameManager.CurrentBattle (Singleton).

Arco Narrativo

"Não existe o melhor estilo de luta — existe o melhor lutador. E o melhor lutador domina todos os estilos."

O protagonista Rui Santos acorda em Braga sem memórias de combate. Percorre Portugal de sul a norte treinando em 10 ginásios. Por trás dos combates existe a Ordem do Cartel — organização que controla o desporto de combate nacional.

No final: o protagonista descobre que já foi campeão, foi derrotado por uma luta corrompida, e os mestres que o treinaram fizeram-no de propósito — para que voltasse sem o peso do ego.

O campeonato final em Lisboa é uma escolha moral. O antagonista Viktor Sousa (Sambo + BJJ) é um antigo companheiro de treino que escolheu o lado da Ordem.
